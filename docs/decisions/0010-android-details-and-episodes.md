# ADR 0010：Android 详情与分集使用现有后端 API

- 状态：Accepted
- 日期：2026-07-20

## 背景

PR #9 只建立服务地址、作品摘要列表和可构建的手机/TV 通用 APK。客户端尚不能读取作品完整元数据和已入库分集，因此无法形成浏览闭环。

后端已经有稳定的正式作品详情和分集接口。此时不需要在 APK 中复制 catalog、抓取来源或创建第二套数据服务。

## 决策

1. 作品摘要继续使用 `GET /api/works`。
2. 打开作品时读取 `GET /api/v1/works/{work_id}` 和 `GET /api/v1/works/{work_id}/episodes`。
3. 客户端校验详情内部 ID、来源作品 ID 及每一集 `work_id`，拒绝跨作品混合响应。
4. 手机使用纵向详情/分集布局；TV 使用详情侧栏和遥控器可聚焦分集网格。
5. 分集点击只显示只读身份和时长；在 production playback provider 与 NAS handoff 可用前不显示播放或下载按钮。
6. 不引入 Navigation、Room、Coil、Media3 或新的网络框架；继续使用当前小型 API 客户端和 ViewModel 状态。

## 原因

- 复用已测试的后端 API，避免 APK 内复制业务数据；
- 内部 ID 与来源 ID 双重校验可以防止错误详情覆盖或跨作品分集混入；
- 一个简单的详情状态足以支撑当前单 Activity 应用；
- 延迟播放器和下载依赖，避免把尚未可用的播放链路伪装成完成。

## 影响

正面：

- 手机与 TV 都能完成作品列表、详情和分集浏览；
- 后续 Media3 可以直接使用稳定的内部 episode ID；
- API JSON 解析和作品/分集身份校验可以通过 JVM 单元测试覆盖。

限制：

- 当前不支持搜索、分页加载更多或收藏历史；
- 点击分集不能播放或下载；
- 未进行手机、盒子或智慧屏真机测试。

# ADR 0006：通过持久化任务刷新公开作品详情和分集标识

- 状态：Accepted
- 日期：2026-07-19

## 背景

全量/增量发现只保存作品级元数据，旧工程则由独立脚本从详情页读取 `vid_list`。Android 和后续 playback API 需要稳定的分集主键，但不能把播放地址、Cookie 或来源会话混入作品目录。

## 决策

1. 新增 `enrich_work` 持久化任务，由来源适配器读取一部作品的公开 SSR 详情。
2. 分集只保存来源分集 ID、集序、标题和可选时长，不保存播放 URL。
3. enrichment 返回的来源名和作品 ID 必须与数据库现有作品一致，否则任务失败。
4. 只有解析到非空 `vid_list` 时才把分集视为权威快照；空或缺失列表保留已有分集。
5. 同一作品同时只允许一个 pending/running enrichment；不同作品可以排队。
6. 正式端点使用内部作品 ID，兼容端点继续接受旧 `series_id`。
7. 管理页仅创建后台任务，不在浏览器直接请求第三方来源。

## 原因

- 作品目录和播放解析保持分层，避免短期 URL 或凭据永久落库；
- 持久化任务让慢网络、失败和进程重启可观察、可 retry；
- 身份校验防止错误来源响应覆盖其他作品；
- 非空快照规则降低来源临时异常造成分集误删的风险。

## 影响

正面：

- 作品详情页可以真实刷新公开元数据和分集；
- APK 后续可以直接按内部作品/分集 ID 浏览；
- enrichment 测试完全使用合成 SSR，不依赖第三方网络。

限制：

- 仍没有播放 URL、headers、expiry 或代理；
- 来源页面结构变化会让任务失败并记录错误；
- 当前只注册 `novelquick` 适配器。
